# easy-agent-mem v0.6.3

## Highlights

- Upgraded graph dashboard readability with visual summary cards in `Index.md`
- Added friendlier and more actionable `--enrich` diagnostics for missing/invalid Groq setup
- Improved large-project graph performance UX with progress output and incremental parse caching
- Expanded README with practical usage flows and a comprehensive command reference

## What Changed

### Graph UX and reporting

- Added dashboard summary cards for code footprint, decision signal, and blocker signal
- Added build performance metrics (cache hits, reparsed files, build duration) to graph report
- Added progress stage emission during graph build for long scans

### Enrich error handling

- Added specific guidance when `GROQ_API_KEY` is missing
- Added clearer handling for missing Groq client package
- Added targeted messaging for auth failures, rate limits, and model errors
- Added CLI warning when `--enrich` was requested but produced no inferred output

### Performance improvements for large repos

- Introduced incremental parse cache at `agent-mem-output/.graph-cache.json`
- Reused unchanged file parse results via file `mtime` and size checks
- Exposed cache hit/miss counters in CLI output

### README updates

- Refreshed feature list to include graph cards, enrich diagnostics, and caching/progress
- Added practical "How To Use" command flows
- Expanded commands section with grouped command reference and examples
- Expanded graph docs with troubleshooting and large-repo guidance
